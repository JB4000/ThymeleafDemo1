Spring & Thymeleaf

This app demonstrates how the MVC pattern is used in Spring Thymeleaf, using a simple user form for data entry and a list of users.

Model - Business Logic & Data

Includes services-, repositories- and entity classes.

Service - Encapsulates business logic

Handles business logic and access data via repositories, and returns a response to the requesting controller.

Repository - Encapsulates data

Handles CRUD: Create, Read, Update, Delete and querying data entities, in a database. Only the repository knows about the database implementation.

Entity - Is the data

Represents a database object (a row in a database table).

View - User Interface (UI)

Includes HTML-, CSS- and JavaScript files.

HTML - Structures the view

Defines the main structure of elements on a web page. The elements are event driven placeholders for content, with attribute for styling, events etc.

CSS - Styles a view

Defines the look and feel style of the page elements. like colors, position, text fonts and styles, animation...

JavaScript - Programmatic interaction with views

A script language used to manipulate the structure and data on a web page and fetch data from external sources/services, via programming.

Controller - Delegates requests to services

Gets incoming web requests from views and response with a result view.

Delegate - Gets requests from a UI view

Validates and delegates the requests to the targeted service.

Response - Returns data from the service request.

Returns a path to a view (html page) and shares data supplied by the service, with the view, by adding view model attributes.

Formatting - Returns data in requested format.

Gets raw data from the service and convert it on the fly to formats like JSON, XML, CSV...
