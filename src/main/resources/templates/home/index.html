Welcome to the Spring Thymeleaf Demo!

This app demonstrates how the MVC pattern is used in Spring Thymeleaf, using a simple user form for data entry and a list of users.

Model - Business Logic & Data

Includes services-, repositories- and entity classes.

Service - Encapsulates business logic
  • Handles business logic and has access to data entities, via repositories, and returns a response to the requesting controller.
Repository - Encapsulates data
  • Handles creating, saving and finding data entities, from a database. Only the repository knows about the database implementation.
Entity - Is the data
  • Represents a database object (a row in a database table).

View - User Interface (UI)

Includes HTML-, CSS- and javaScript files.

HTML - Structures the view
  • Defines the main structure of elements on a web page.
CSS - Styles a view
  • Defines the look and feel style of the page elements.
Javascript - Programmatic interaction with views
  • A script language used to manipulate the structure and data on a web page and fetch data from external sources/services, via programming.

Controller - Links UI request to business logic and data responses

Gets incoming web requests from views and response with a result view.

Delegate - Gets requests from a UI view
  • Validates and delegates the requests to the targeted service.
Response - Gets response data from their controller request
  • Returns a path to a proper view (html page) and shares, data supplied by the service, with the view, by adding view model attributes.
